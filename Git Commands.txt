Check Git is installed
************************
			git --version


configure git username and email
**************************************
		git config --global user.name "Your Name"
		git config --global user.email "your@email.com"

Verifying the credentials:
****************************
		git config --global --list



Intialize a Git repository:
***************************
		git init


Connect local repo to remote (GitHub / GitLab)
****************************************************
		git remote add origin https://github.com/username/repository.git

Verify remote
***************
		 git remote -v


Create a branch
*******************
		git branch branchname

Switch back to another branch
***********************************
		git checkout main
			(or)
		git switch main

Delete branch locally
*************************
		git branch -D branchname


Check the branches:
***********************
		git branch





Git is a distributed version control system used to track changes in source code and help multiple developers work together.

In Simple Words

“Git remembers every change in your code and lets you go back anytime.


There are 3 Main Areas in Git:
1. working directory
2. Staging Area
3. Repository

1. working directory
  where you write and edit code
  normal folder on the computer


2. staging area

 it is a temporary  area
 we select what changes need to be saved

3. repository

where git permanently stores changes
contains full history


Basic Git Commands Revision(for Devops)

1. git init
creates a new git repository

**********    git init    **********

Real life: start tracking this folder



2. git status
shows current state of files

**********    git status    **********

Tells us the modified files,staged files,untracked files


3. git add
moves files to staging area


git add filename.py
git add .


4. git commit
Saves Changes to repository with a message

git commit -m "Added Features"


5. git log
shows commit history

git log


6. git diff
Shows what changed

git diff

Git branching:
***************
A branch in git is a separate line of development that lets you work on features or fixes without affecting the main code


A branch is a safe copy of the project where you can experiment

Real life example:
	rough notebook in exam is branch
	main notebook is the main branch

1. git branch
shows all branches

**  git branch


2. git branch <name>
create  a new branch


**  git branch branchname

created but not switched

3. git checkout <branch>
switch to another branch

**  git checkout branchname


4. git checkout -b <branch>

create + switch in one step 

Important
***********     git checkout -b branchname


5, git merge
merge branch into current branch

*****     git checkout main
	    git merge branchname
